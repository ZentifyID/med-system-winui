# Med System (WinUI 3)

Настольное приложение для медпункта колледжа: учёт обращений, данных
сотрудников и студентов, лекарств со сроками годности и справочником МКБ-10.
Переписано с Python/tkinter на C# / WinUI 3.

> Схема БД полностью совместима с python-версией
> ([med-system](https://github.com/ZentifyID/med-system)) — обе программы
> открывают один и тот же файл `med_system.db`.

## Возможности

- **Сотрудники и студенты** — карточки с медицинскими сроками (санминимум,
  медосмотр, флюорография); просроченные сроки подсвечиваются бейджами.
- **Учебные группы** — списки студентов, перевод группы на следующий курс.
- **Лекарства** — остатки и сроки годности, контроль просрочки, заказ партий.
- **Журнал обращений** — автодополнение отправителя и диагнозов по
  встроенному справочнику МКБ-10 (34 частых кода засеиваются при первом запуске).
- **Справочник МКБ** — поиск по коду и названию.

## Технологии

| Слой | Стек |
|------|------|
| UI | WinUI 3, Windows App SDK 2.2.0 |
| Рантайм | .NET 8 (`net8.0-windows10.0.19041.0`) |
| Данные | SQLite через `Microsoft.Data.Sqlite` |
| Упаковка | MSIX (single-project packaging) |

**Минимальная ОС:** Windows 10 версии 1809 (10.0.17763). Поддерживаемые
архитектуры: x64, x86, ARM64.

## Структура решения

```
MedSystem/
├── MedSystem.Core/    Домен: модели, правила сроков, валидация (без зависимостей)
├── MedSystem.Data/    Слой данных: SQLite, репозитории, инициализация схемы
└── MedSystem.App/     Интерфейс: WinUI 3, страницы и формы
```

Зависимости слоёв: `App → Data → Core`. `Core` и `Data` — обычные библиотеки
`net8.0`, весь Windows-специфичный код изолирован в `App`.

## База данных

SQLite, файл `med_system.db`. Расположение зависит от режима запуска:

- **Упакованное (MSIX) приложение** — в `LocalFolder` пользователя:
  `%LOCALAPPDATA%\Packages\<PackageFamilyName>\LocalState\med_system.db`.
  Путь переопределяется в [`App.xaml.cs`](MedSystem/MedSystem.App/App.xaml.cs)
  при старте.
- **По умолчанию** (`Db.DbPath`) — рядом с исполняемым файлом.

Схема создаётся автоматически при первом запуске
([`DatabaseInitializer`](MedSystem/MedSystem.Data/DatabaseInitializer.cs)),
миграции не требуются.

## Сборка и запуск (для разработки)

Требуется **Visual Studio 2022** с рабочей нагрузкой
«Разработка приложений для Windows» (Windows App SDK C# Templates).

1. Открыть `MedSystem/MedSystem.slnx`.
2. Выбрать конфигурацию платформы (`x64` для обычного ПК).
3. Нажать **F5** — NuGet-пакеты подтянутся автоматически.

Из командной строки:

```powershell
dotnet build MedSystem/MedSystem.slnx -c Debug
```

## Перенос приложения на другой ПК

Приложение **упакованное (MSIX)** и хранит базу в `LocalFolder`, поэтому
просто скопировать `.exe` в другую папку нельзя — нужен установочный
MSIX-пакет. Ниже — полный путь от сборки до запуска на чистом компьютере.

### Шаг 1. Собрать установочный пакет (на своём ПК)

**Через Visual Studio (проще всего):**

1. ПКМ по проекту `MedSystem.App` → **Package and Publish → Create App Packages…**
2. Выбрать **Sideloading** (не Microsoft Store).
3. На шаге сертификата — **Create** (создастся самоподписанный сертификат
   `CN=zgami`) или выбрать существующий `.pfx`. Запомни/запиши пароль.
4. Отметить нужные архитектуры (обычно достаточно **x64**), Bundle — по вкусу.
5. Готовый пакет появится в
   `MedSystem/MedSystem.App/AppPackages/…` — там будут `.msix`/`.msixbundle`,
   сертификат `.cer` и скрипт `Install.ps1`.

**Через командную строку (то же самое):**

```powershell
cd MedSystem/MedSystem.App
dotnet publish -c Release -p:Platform=x64
# либо MSIX-пакет с публикацией профиля win-x64
```
Профили публикации в `Properties/PublishProfiles/` собраны с `SelfContained=true`,
поэтому **.NET-рантайм устанавливать на целевой ПК не нужно** — он идёт внутри.

### Шаг 2. Перенести пакет на целевой ПК

Скопируй всю папку `AppPackages/MedSystem.App_<версия>_Test` (флешка, сеть,
облако — как угодно). Внутри лежат три важные вещи:

- `MedSystem.App_<версия>_<арх>.msix` (или `.msixbundle`) — сам пакет;
- `MedSystem.App_<версия>.cer` — сертификат;
- `Install.ps1` — скрипт установки.

### Шаг 3. Установить на целевом ПК

MSIX от самоподписанного сертификата не установится, пока сертификат не
доверенный. Два способа:

**A. Скриптом (рекомендуется):**
ПКМ по `Install.ps1` → **Выполнить с помощью PowerShell**. Скрипт сам
установит сертификат и пакет. Если PowerShell блокирует запуск:

```powershell
# запустить PowerShell от администратора в папке пакета
Set-ExecutionPolicy -Scope Process Bypass
.\Install.ps1
```

**B. Вручную:**
1. Установить сертификат (один раз, от администратора):
   ПКМ по `.cer` → **Установить сертификат** → **Локальный компьютер** →
   поместить в **Доверенные корневые центры сертификации**.
   Или командой:
   ```powershell
   Import-Certificate -FilePath ".\MedSystem.App_<версия>.cer" `
     -CertStoreLocation Cert:\LocalMachine\Root
   ```
2. Двойной клик по `.msix` → **Установить**.

После установки приложение появится в меню «Пуск» как **MedSystem.App**.

### Шаг 4. Перенос данных (по необходимости)

Если нужно перенести и накопленную базу, а не начинать с пустой:

1. На старом ПК найди файл БД:
   `%LOCALAPPDATA%\Packages\<PackageFamilyName>\LocalState\med_system.db`
   (`PackageFamilyName` виден в приложении на главной странице / в
   `Get-AppxPackage *MedSystem*`).
2. Скопируй `med_system.db`.
3. На новом ПК **сначала запусти приложение один раз** (чтобы создалась
   папка `LocalState`), закрой его, затем положи файл в тот же путь,
   заменив пустую базу.

### Обновление версии

Подними `Version` в
[`Package.appxmanifest`](MedSystem/MedSystem.App/Package.appxmanifest)
(например `1.0.0.0 → 1.0.1.0`), пересобери пакет тем же сертификатом и
установи поверх — данные в `LocalState` сохранятся.

### Частые проблемы

| Симптом | Причина / решение |
|---------|-------------------|
| «Не удалось установить пакет» / ошибка доверия | Сертификат `.cer` не установлен в **Доверенные корневые** на целевом ПК (Шаг 3). |
| «This app package's publisher certificate could not be verified» | Тот же корень — сертификат не доверенный либо истёк. |
| Приложение не запускается, требует рантайм | Пакет собран без `SelfContained` — пересобери по профилю из `PublishProfiles/`. |
| Установка заблокирована политикой | Включи режим разработчика или sideloading: *Параметры → Конфиденциальность и защита → Для разработчиков*. |
| Не та архитектура | Собери пакет под архитектуру целевого ПК (x64 для большинства, ARM64 для Windows on ARM). |
